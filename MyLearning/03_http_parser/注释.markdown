`http_coon.h`
`🪓 HTTP 解析的“三板斧”`

### 第一板斧：切菜刀 (parse_line)
任务：在一长串字符里找到 \r\n，把它切断。 原理： HTTP 协议规定，每一句话后面都要跟一个回车换行符 (\r\n)。 
比如：GET / HTTP/1.1\r\n。 我们的“切菜刀”就是遍历数组，看到 \r 且下一个是 \n，就把它们改成字符串结束符 \0。 
潜台词：“这一行读完了，截断！”

### 第二板斧：信号灯 (enum State)
任务：记住现在读到第几行了。 原理： HTTP 请求像个三明治：
面包盖 (Request Line)：GET / HTTP/1.1 (最关键的第一行)
夹心菜 (Headers)：Host: localhost (各种参数)
面包底 (Content)：(有时候没有)
我们需要一个变量（信号灯）来告诉程序：“现在正在吃面包盖” 还是 “现在正在吃夹心菜”。

### 第三板斧：分拣机 (Switch Loop)
任务：根据信号灯，决定怎么处理这一行。 原理：
如果是“面包盖” -> 提取 URL。
如果是“夹心菜” -> 看看有没有 Keep-Alive。
如果是“空行” -> 说明吃完了！

`http_conn.cpp`
1、🧱 基础设施篇 (Infrastructure)
内容：头文件引用、静态变量初始化、Epoll 的辅助函数（设置非阻塞、注册事件、删除事件）。
难度：⭐⭐
作用：这是给 Epoll 打下手的工具箱。

2、🚪 连接管理篇 (Connection Mgmt)
内容：init (初始化)、close_conn (关闭连接)。
难度：⭐⭐
作用：负责“迎来送往”，可以编译通过但还没法说话。

3、👂👄 IO 读写篇 (Input/Output)
内容：read_once (读)、write (写)。
难度：⭐⭐⭐
作用：真正和 TCP 缓冲区打交道的地方。

4、🧠 逻辑解析篇 (Logic Parser)
内容：process_read (主状态机)、parse_xxx (各种解析)。
难度：⭐⭐⭐⭐⭐ (最核心、最复杂)
作用：把乱码变成 HTTP 请求对象。

5、📦 响应生成篇 (Response Builder)
内容：process_write (总控)、add_xxx (拼凑报文)。
难度：⭐⭐⭐⭐
作用：把结果打包发回去。
